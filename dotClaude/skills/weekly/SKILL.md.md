# Weekly Review

## Instructions
Read ALL daily notes from this week (Monday-Sunday).
Read friction-log/ entries from this week.
Read habits/metrics-log.md for this week's data.
Read now/ folder for project status changes.

## Pattern detection — look specifically for:
- Repeated missed tasks (same task appearing multiple days)
- Time-of-day performance patterns (morning vs evening execution)
- Overplanning vs execution gaps (planned 10 tasks, did 4)
- Avoidance behavior (same task deferred 3+ times)
- Energy vs productivity correlation (low energy days vs execution %)
- Friction patterns (same type of friction appearing repeatedly)

## Reality gap check:
Calculate: total tasks planned this week vs total tasks completed
Identify: which categories had the worst completion rate (farm/build/school/people/faith)

## Effort estimation check (if logged):
Compare estimated vs actual time on tasks where both were recorded

## Output FORMAT — enforce strictly:
```

## Weekly Review — Week of [DATE]

**Reality Gap** Planned: [X] tasks · Completed: [Y] tasks · Gap: [Z]% Worst category: [category] ([X]% completion)

## What Worked

[2-4 bullet points, specific]

## What Didn't

[2-4 bullet points, specific]

## Patterns Detected

[2-4 bullet points with confidence tags] 🟢/🟡/🔴 [pattern description]

## Likely Causes

[Brief explanation of root causes — not surface symptoms]

## Friction Summary

## Suggested Adjustments

[2-3 specific, actionable changes — not vague advice]

```

## Rules
- Confidence tags mandatory on all pattern observations
- Save to daily/<current year ie.2026>/YYYY-WW-weekly.md
- Update now/ folder with any project status changes needed
- Flag any upcoming deadlines in the next 14 days
